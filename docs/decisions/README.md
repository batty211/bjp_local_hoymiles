# Architecture Decision Records

Use ADRs to record decisions that materially affect architecture, operations,
security, data compatibility, or long-term maintenance.

## Naming

Use sequential names:

```text
0001-short-decision-title.md
0002-next-decision.md
```

## Status Values

- Proposed
- Accepted
- Superseded by ADR-NNNN
- Rejected

Copy `0000-template.md` when creating a new decision. Do not rewrite accepted
decision history; create a new ADR that supersedes it.
